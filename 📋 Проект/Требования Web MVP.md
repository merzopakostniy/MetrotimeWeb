# Требования Web MVP

Дата создания: 2026-04-26

## Цель MVP

Сделать мобильную web-версию МетроТайм, которой можно пользоваться с телефона через браузер, с дальнейшим подключением Premium-подписки.

## Принципы

- Mobile-first.
- Быстрый доступ без App Store.
- Понятный интерфейс для ежедневного использования.
- Данные пользователя должны быть защищены.
- Расчеты должны совпадать с iOS-версией или иметь явно зафиксированные отличия.
- Все решения фиксируются в Obsidian.

## MVP-функции

### Аккаунт

- Регистрация.
- Вход.
- Выход.
- Восстановление пароля.
- Профиль пользователя.

### Смены

- Добавление смены.
- Редактирование смены.
- Удаление смены.
- Список смен по месяцам.
- Быстрый переход к текущей дате.
- Заметки к смене.
- Автоподсказки начальной и конечной точки.

### Расчеты

- Разбивка часов на дневные, вечерние и ночные.
- Базовый заработок по тарифу.
- Учет праздников.
- Учет роли Машинист/ТЧМ.
- Учет основных флагов смен.
- Месячная норма часов.
- Годовая статистика.

### Настройки

- Роль пользователя.
- Тарифная ставка.
- Дата трудоустройства.
- Класс квалификации.
- Наставник.
- Профсоюз.
- Алименты.

### Premium

- Экран тарифов.
- Статус подписки.
- Ограничение premium-функций.
- Серверная проверка подписки.
- История платежных событий через webhooks.

## Не входит в первый MVP

- Apple Watch.
- iOS widgets.
- iCloud Drive backup.
- Сложная миграция из iOS без отдельного плана.
- Админ-панель, если она не нужна для запуска оплат.

## Критерии готовности MVP

- Пользователь может создать аккаунт и войти.
- Пользователь может вести смены с телефона.
- Расчеты по тестовым сценариям совпадают с ожидаемыми значениями.
- Данные сохраняются после перезагрузки и доступны после повторного входа.
- Есть базовая premium-модель и понятный путь подключения платежей.
- Основные решения и открытые вопросы записаны в Obsidian.
