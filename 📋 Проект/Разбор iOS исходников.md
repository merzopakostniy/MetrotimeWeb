## Факты, которые нельзя потерять

- Смена сохраняется через upsert по календарному дню: существующие смены за тот же день удаляются.
- Массовый upsert также удаляет смены, которые пересекают заменяемый день.
- После каждого изменения пересчитываются:
  - `baseEarnings`;
  - `nextShiftStartTime`;
  - `breakBetweenShifts`;
  - `evenOddFlag`;
  - справочники начальных/конечных точек.
- Заметки хранятся и в `Shift.notes`, и отдельно в `UserDefaults` по ключу `shiftNote_<UUID>`.
- Список локаций хранится отдельно: `startLocationsKey`, `endLocationsKey`.
- Для ТЧМ в списке создаются виртуальные карточки продолжения ночных смен на следующий календарный день. База данных при этом не меняется.
- Firestore в iOS хранит весь `shifts.json` в `users/{uid}/meta/shiftsDoc.json`.
- Auth: Apple-пользователь считается вошедшим сразу; email-пользователь - только после `isEmailVerified`.

## Правила роли

В iOS роли уже enum:
- `driver` - Машинист;
- `tcm` - ТЧМ;
- `dde` - ДДЭ;
- `sec` - Служба безопасности;
- `station` - Дежурный по станции.

В web нужно хранить enum-значение стабильно, а русский текст считать отображением.

## UX деталей из iOS

- Главная навигация: список смен, сегодня, норма, статистика, настройки.
- Список смен группируется по месяцам.
- Есть поиск по дате и автоскролл к сегодня.
- На карточке смены показываются дата, день недели, название, время, точки, флаги и индикатор заметки.
- Форма добавления содержит подсказки шаблонов смен из истории.
- Спецдни отключают часть рабочих полей.
- Некоторые спецфлаги взаимно исключаются: отпуск, учебный отпуск, больничный, больничный по уходу, донорский день, медкомиссия.

## Web-выводы

- Сначала переносить доменную логику, а не визуальные эффекты.
- Расчеты вынести в тестируемый модуль.
- Схему БД делать нормализованной, не одним JSON-документом.
- Подписки Premium проектировать серверно с webhook.
- Для совместимости с iOS предусмотреть импорт старого JSON.
