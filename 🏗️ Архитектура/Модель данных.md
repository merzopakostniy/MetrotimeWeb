# Модель данных

Статус: черновик на основе iOS-исходников.

## Shift

Источник: `MetrotimeIOS/Metrotime/Shift.swift`.

### Основные поля

| Поле iOS | Тип iOS | Web-смысл |
|---|---|---|
| `id` | UUID | ID смены |
| `date` | Date | календарная дата смены |
| `name` | String | название смены |
| `startTime` | Date | время начала |
| `startPoint` | String | место/ЛП заступления |
| `endTime` | Date | время окончания |
| `endPoint` | String | место/ЛП завершения |
| `dateRange` | DateRange? | диапазон для спецдней |
| `notes` | String | заметка |
| `imageData` | Data? | вложенное изображение, для web пока под вопросом |
| `nextShiftStartTime` | Date? | начало следующей смены |
| `evenOddFlag` | String? | `ЧЕТ` / `НЕЧ` для непрерывной ночи |
| `baseEarnings` | Double | базовый заработок |
| `mentorBonus` | Double | в модели дефолт 500, но расчет нормы использует 15% |
| `breakBetweenShifts` | TimeInterval | перерыв до следующей смены |

- `isActingSM` - и. о. СМ;

Типы смен/дней:
- `isDayOff` - выходной;
- `isReserveShift` - резерв;
- `isSurfaceReserve` - резерв на поверхности;
- `isVacation` - отпуск;
- `isStudyLeave` - учебный отпуск;
- `isSickLeave` - больничный;
- `isSickLeaveCare` - больничный по уходу;
- `isDonorDay` - донорский день;
- `isInternalTraining` - внутреннее обучение;
- `isMedicalCheckup` - медкомиссия.

## Производные свойства

### `adjustedStartTime`

Комбинирует дату смены и время начала.

### `adjustedEndTime`

Комбинирует дату смены и время окончания. Если окончание меньше или равно началу, окончание переносится на следующий день.

### `isWorkingShift`

`false`, если смена является отпуском, выходным, больничным, медкомиссией, учебным отпуском, больничным по уходу, донорским днем или внутренним обучением.

### `isSpecialShift`

`true` для выходного, больничного, отпуска, медкомиссии, учебного отпуска, больничного по уходу, донорского дня или внутреннего обучения.

## Правило уникальности

В iOS одна смена на календарный день. `upsertShift` удаляет все смены на тот же день и добавляет новую.

Для web это нужно подтвердить как бизнес-правило до реализации БД. Если в будущем нужны две смены в день, это будет отдельное изменение продукта.
